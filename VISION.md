## 1. 项目初衷

### 1.1 核心价值主张

打造一款**高度个性化的英语单词学习桌面应用**，通过智能算法为每位用户定制最优学习路径，让单词记忆更高效、更持久。

### 1.2 目标用户

| 用户群体 | 核心需求 | 学习特点 |
|----------|----------|----------|
| **中小学生** | 课本同步词汇、应试提分 | 记忆周期短，需要高频强化 |
| **四六级考生** | 大量词汇突击、冲刺复习 | 时间紧迫，需要高效筛选 |

### 1.3 解决的核心痛点

- **盲目刷词**：不知道哪些词该复习，哪些已掌握
- **千篇一律**：所有人按相同顺序学习，忽略个体差异
- **效率低下**：大量时间浪费在已掌握词汇上
- **缺乏反馈**：不知道自己的真实水平和进步情况

## 2. 核心功能定义

### 2.1 个性化引擎（四大支柱）

| 支柱 | 技术实现 | 用户价值 |
|------|----------|----------|
| **智能推荐** | 艾宾浩斯遗忘曲线 + 间隔重复算法（SRS） | 在即将遗忘的临界点精准推送复习 |
| **个性化内容** | 内置词库 + 用户自定义导入 | 支持课本同步、行业术语、考试真题词表 |
| **难度自适应** | 动态难度评估模型 | 根据用户表现自动调整词汇难度梯度 |
| **学习风格适配** | 多模态记忆模式（视觉/听觉/情境） | 支持图片、发音、例句、联想记忆 |

### 2.2 功能优先级矩阵

#### P0 - MVP 核心功能（第一版）

| 功能模块 | 具体内容 | 用户价值 |
|----------|----------|----------|
| **内容卡片展示** | 单词 + 音标 + 释义 + 例句 | 清晰呈现学习内容 |
| **记忆状态标记** | 认识/模糊/不认识（3态标记） | 快速记录学习反馈 |
| **错题本** | 自动收集"模糊"+"不认识"的词 | 集中攻克薄弱点 |
| **生词本** | 手动添加重点词汇 | 个性化积累本 |
| **测试功能** | 单选/拼写/听写模式 | 检验学习效果 |
| **词库管理** | 切换词库、查看词库详情 | 灵活切换学习内容 |

#### P1 - 增强功能（第二版）

| 功能模块 | 具体内容 | 用户价值 |
|----------|----------|----------|
| **智能复习调度** | SM-2 算法实现的间隔重复 | 自动安排今日复习词 |
| **学习进度统计** | 可视化图表（学习曲线、掌握率） | 直观看到进步轨迹 |
| **复习提醒** | 桌面通知/定时提醒 | 养成每日学习习惯 |
| **发音练习** | TTS 文字转语音 | 支持听力跟读 |

## 3. 技术可行性评估

### 3.1 桌面应用技术选型

| 技术栈 | 优势 | 劣势 | 推荐指数 |
|--------|------|------|----------|
| **Electron + React/Vue** | 跨平台、生态丰富、UI 灵活 | 体积大、资源占用高 | ⭐⭐⭐ |
| **PyQt / PySide** (Python) | 轻量、Python 生态、算法友好 | UI 开发复杂度较高 | ⭐⭐⭐⭐⭐ |
| **Tkinter** (Python) | 内置库、零依赖 | UI 简陋、扩展性差 | ⭐⭐ |

**推荐方案**: **PyQt6/PySide6** + Python
- 算法实现友好（SRS、数据分析）
- 足够现代的 UI 能力
- 轻量级部署

### 3.2 核心算法可行性

#### SM-2 间隔重复算法

```
基础公式（SuperMemo 2）：

I(1) = 1
I(n) = I(n-1) × EF

其中：
- I(n): 第 n 次复习的间隔天数
- EF (Easiness Factor): 易度因子，初始值 2.5
- EF 根据用户评分动态调整
```

**可行性**: ✓ 算法成熟，已有大量成功案例（Anki、SuperMemo）

#### 难度自适应模型

```
用户能力估算（简化 ELO 机制）：

Ability_new = Ability_old + K × (Actual - Expected)

其中：
- K: 学习速率参数
- Actual: 实际表现（1=正确，0=错误）
- Expected: 预期正确率（基于词汇难度）
```

**可行性**: ✓ 经典心理测量模型，适合自适应学习

### 3.3 数据存储方案

| 数据类型 | 存储方案 | 理由 |
|----------|----------|------|
| 用户学习记录 | SQLite | 本地存储、事务支持、零配置 |
| 内置词库 | JSON/CSV | 轻量、易扩展、可版本控制 |
| 用户导入词库 | CSV/TXT | 通用格式、用户友好 |

## 4. MVP 范围界定

### 4.1 MVP 包含（P0）

```
englishstudy/
├── data/
│   ├── vocab/          # 内置词库（JSON格式）
│   │   ├── middle_school.json
│   │   ├── high_school.json
│   │   └── cet4.json
│   └── user/           # 用户数据目录
│       └── study.db    # SQLite 学习记录
├── src/
│   ├── core/           # 核心算法
│   │   ├── srs.py      # 间隔重复算法
│   │   └── difficulty.py # 难度自适应
│   ├── models/         # 数据模型
│   ├── ui/             # PyQt 界面
│   └── utils/          # 工具函数
└── main.py             # 应用入口
```

### 4.2 MVP 明确不包含

- 云端同步功能
- 社交分享功能
- 在线词典 API 集成
- 移动端适配

## 5. 技术风险预警

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| PyQt 学习曲线 | UI 开发周期可能延长 | 优先使用 Qt Designer 原型设计 |
| SRS 算法调优 | 复习间隔可能不够精准 | MVP 阶段使用简化版 SM-2，P1 优化 |
| 词库版权问题 | 内置词库来源需合规 | 使用开源词库或自建词库 |

## 6. 成功指标（MVP 验收标准）

- [ ] 用户能成功导入词库或使用内置词库
- [ ] 能进行单词卡片学习并标记记忆状态
- [ ] 错题本、生词本能正确收集词汇
- [ ] 测试功能能生成随机测试题并给出评分
- [ ] 学习记录能持久化保存
- [ ] 应用能在 Windows/macOS/Linux 上正常运行
